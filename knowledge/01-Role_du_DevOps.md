# 01: Rôle du DevOps & Collaboration

## 1. Définition du Rôle
Le DevOps n'est pas seulement un expert des outils (CI/CD, Docker, K8s), c'est un "facilitateur" qui comble le fossé entre le Développement (Dev) et les Opérations (Ops).
*   **Objectif Principal** : Accélérer la livraison de logiciel (Speed) tout en maintenant la stabilité (Reliability).
*   **DevOps vs Cloud Engineer** :
    *   *Cloud Engineer* : Focus sur l'infrastructure (le "où" ça tourne).
    *   *DevOps Engineer* : Focus sur le processus et l'automatisation (le "comment" ça arrive en prod).

## 2. Compétences Clés (Niveau Senior)
Pour dépasser le niveau junior, il faut maîtriser le "Savoir-Être" (Soft Skills) autant que le technique.
*   **"Manager of One"** : Concept clé de *DevOpsRoadmap.io*. C'est la capacité à fixer ses propres objectifs et à les exécuter sans supervision constante. Un senior n'attend pas qu'on lui donne un ticket, il identifie le besoin.
*   **T-Shaped Skills** :
    *   Barre verticale (Profondeur) : Expertise forte dans un domaine (ex: CI/CD ou K8s).
    *   Barre horizontale (Largeur) : Compréhension globale (Dev, QA, Sécurité, Business) pour collaborer avec tous.

## 3. Collaboration & Culture (CALMS)
Le modèle **CALMS** est la référence pour la culture DevOps :
*   **C**ulture : Responsabilité partagée. "You build it, you run it". Pas de blâme (Blameless culture).
*   **A**utomation : Éliminer le "Toil" (tâches manuelles répétitives) pour se concentrer sur la valeur.
*   **L**ean : Minimiser le gaspillage (attente, code non utilisé).
*   **M**easurement : On ne peut améliorer que ce qu'on mesure (Métriques DORA : Fréquence de déploiement, Lead Time, MTTR, Taux d'échec).
*   **S**haring : Documenter et partager les connaissances (pour éviter d'être le seul sachant = "Bus Factor").

## 4. Pratiques de Collaboration
*   **Shift Left** : Impliquer la sécurité et les ops tôt dans le cycle de dév.
*   **Communication Asynchrone** : Privilégier l'écrit (Tickets, Docs) aux réunions pour laisser des traces.
*   **Code Reviews** : Ne pas juste corriger le code, mais mentorer les juniors via les commentaires.
